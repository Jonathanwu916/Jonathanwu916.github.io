OOP in Javacript(2)

3 ways to implement inheritance

1. Prototype Chain way

The biggist disvantage of prototype chain way is cannot call bases’ constructor in child class. As we cannot assign property value from derived to base.
The second one is if changing the reference type data property on first instance of derived class, the others has been modified at the same time.

We normally do not use this way to implement inheritance.

2. Using call

In this way, every objects has the say function. It takes too many spaces. So we may not use this way either.

3. Complex way

We inherit property through call method, and inherit method through prototype chain. In think which is the best way to implement inheritance.

In Conclusion

Above cases are the ways to inherit and apparently last one is the best way. It is interesting to implement OOP in javascript. Because there are always mutiple ways to learn. It means you are more likely to create amazing things.
