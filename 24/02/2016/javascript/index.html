OOP in Javacript(1)

8 ways to create object

1. Simple way

Since there is no class in javascript. We can create instance through object directly. For this way, problem is reusing the instance because of no class.

2. Json

Obviously, it has same problem with previous way.

3. Factory

Through this way, we solve the class problem. But new problem comes out - we cannot detect the data type of p1 and p2.

4. Constructor

Because the say function is global, window can call say() as well. It means the encapsulation here is bad. For another, if there is a object contains a lot of fucntions, it causing hundreds of global function come into project. Which is not good for development.

5. Prototype

This way achive encapsulation for object. Window cannot call say() any more. But in this case, the constructor will assign to object instead of Person.

6. Prototype Json
